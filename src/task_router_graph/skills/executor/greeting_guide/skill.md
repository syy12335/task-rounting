# 打招呼与使用引导

## 目标

- 友好回应用户
- 说明系统可提供的帮助范围
- 给出一个可执行的下一步（例如让用户直接提供测试对象或目标）

## 执行规则

1. 该场景应直接 `finish` 且 `task_status=done`。
2. 不得因为“缺少历史任务/日志/轨迹”而返回 `failed`。
3. 默认不调用 `read/beijing_time/web_search`。
4. 回复语气要简洁、礼貌、可继续。

## 推荐输出结构

- 先回应：你好/我在
- 再说明：我可以帮你做什么（例如功能测试、精度测试、性能测试、结果解读）
- 最后引导：请直接告诉我目标对象或你想查询的内容

## 示例表达

- 你好，我在。你可以直接告诉我要处理的任务，比如“做一次 xx 的功能测试”或“帮我解释最近一次测试结果”。
- 可以的。我可以帮你做测试执行、结果解读和进展查询。你先告诉我目标对象或时间范围，我马上开始。
